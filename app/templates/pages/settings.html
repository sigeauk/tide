Sigma Pipeline Guide

A Processing Pipeline transforms generic Sigma rules into backend-specific queries (e.g., Elasticsearch DSL). It consists of three primary components:

  • Identifer & Priority: Every pipeline needs a unique name. The priority determines the order in which multiple pipelines are applied.
  • Field Mapping: Use the field_name_mapping transformation to translate Sigma's standard field names (like Image) to your environment's schema (like process.executable).
  • Index Scoping: For Elasticsearch detection-rule output, use backend index_names (from the Sigma page index field) or a template pipeline vars.index_names. This follows native pySigma/sigma-cli behavior.
Pro Tip: Use both selectors on the Sigma Converter page: Custom Pipelines for field mapping and Templates for postprocessing output shaping.

External API Keys

Manage API keys for the external query endpoint (/api/external/query). Keys authenticate external applications to run read-only SQL queries against TIDE data.

External Query API (HTTPS)

Endpoint: POST /api/external/query

Header: X-TIDE-API-KEY: <your key>

Request body: {"sql": "SELECT ..."}

Security constraints
  • Only SELECT and WITH (CTE) statements are allowed
  • Blocked keywords include DROP, DELETE, INSERT, UPDATE, INSTALL, and other mutating/admin operations
  • Maximum SQL length: 4000 characters
  • Keys are stored as SHA-256 hashes; raw key is shown only once at creation
Common query tables

detection_rules, threat_actors, mitre_techniques, systems, hosts, software_inventory, vuln_detections, applied_detections, blind_spots, playbooks, playbook_steps, step_techniques, step_detections, system_baselines, system_baseline_snapshots, classifications, app_settings

cURL examples and responses

Count all rules

curl -s -X POST https://your-tide-host/api/external/query \
  -H "Content-Type: application/json" \
  -H "X-TIDE-API-KEY: YOUR_KEY_HERE" \
  -d '{"sql":"SELECT COUNT(*) AS total_rules FROM detection_rules"}'
{"columns":["total_rules"],"rows":[{"total_rules":239}],"row_count":1}

List production rules by score

curl -s -X POST https://your-tide-host/api/external/query \
  -H "Content-Type: application/json" \
  -H "X-TIDE-API-KEY: YOUR_KEY_HERE" \
  -d '{"sql":"SELECT rule_id, name, severity, score FROM detection_rules WHERE space = '\''production'\'' ORDER BY score DESC LIMIT 5"}'

Error examples

401: {"detail":"Invalid or missing API key."}
400: {"detail":"Only SELECT statements are allowed."}
400: {"detail":"Forbidden SQL keyword: DROP"}
