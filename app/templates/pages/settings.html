Sigma Pipeline Guide

A Processing Pipeline transforms generic Sigma rules into backend-specific queries (e.g., Elasticsearch DSL). It consists of three primary components:

  • Identifer & Priority: Every pipeline needs a unique name. The priority determines the order in which multiple pipelines are applied.
  • Field Mapping: Use the field_name_mapping transformation to translate Sigma's standard field names (like Image) to your environment's schema (like process.executable).
  • Index Scoping: Use add_condition to automatically append index constraints (e.g., index: logs-*) to every rule converted using this pipeline.
Pro Tip: You can select these pipelines directly from the Sigma Converter page via the "Custom Pipeline" dropdown to apply your mappings on-the-fly.
