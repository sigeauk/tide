API Key Management

Manage API keys for the external query endpoint (/api/external/query). Keys authenticate external applications to run read-only SQL queries against TIDE data.

External Query API (HTTPS)

Query endpoint: POST /api/external/query

List tenants: GET /api/external/clients

Header: X-TIDE-API-KEY: <your key>

Request body: {"sql": "SELECT ...", "client_id": "TENANT_ID"}

The API key inherits your tenant access. If you have access to one tenant, client_id is optional. Use /api/external/clients to discover your available tenants.

Security constraints
  • Only SELECT and WITH (CTE) statements are allowed
  • Blocked keywords include DROP, DELETE, INSERT, UPDATE, INSTALL, and other mutating/admin operations
  • Maximum SQL length: 4000 characters
  • Queries run against isolated tenant databases — each API key can only access tenants assigned to its owner
  • Tenant DB is opened read-only; no writes are possible even if SQL validation were bypassed
  • Keys are stored as SHA-256 hashes; raw key is shown only once at creation
Common query tables

detection_rules, threat_actors, mitre_techniques, systems, hosts, software_inventory, vuln_detections, applied_detections, blind_spots, playbooks, playbook_steps, step_techniques, step_detections, system_baselines, system_baseline_snapshots, classifications, app_settings

cURL examples and responses

List your tenants

curl -s https://your-tide-host/api/external/clients \
  -H "X-TIDE-API-KEY: YOUR_KEY_HERE"
{"clients":[{"id":"1b650e71-...","name":"Primary Client","slug":"primary"}]}

Count all rules (single tenant — client_id optional)

curl -s -X POST https://your-tide-host/api/external/query \
  -H "Content-Type: application/json" \
  -H "X-TIDE-API-KEY: YOUR_KEY_HERE" \
  -d '{"sql":"SELECT COUNT(*) AS total_rules FROM detection_rules"}'
{"columns":["total_rules"],"rows":[{"total_rules":239}],"row_count":1}

Query a specific tenant

curl -s -X POST https://your-tide-host/api/external/query \
  -H "Content-Type: application/json" \
  -H "X-TIDE-API-KEY: YOUR_KEY_HERE" \
  -d '{"sql":"SELECT rule_id, name, severity, score FROM detection_rules ORDER BY score DESC LIMIT 5", "client_id":"1b650e71-..."}'

Error examples

401: {"detail":"Invalid or missing API key."}
400: {"detail":"Only SELECT statements are allowed."}
400: {"detail":"client_id is required when the API key owner has access to multiple tenants..."}
403: {"detail":"API key does not have access to the requested tenant."}
400: {"detail":"Forbidden SQL keyword: DROP"}
